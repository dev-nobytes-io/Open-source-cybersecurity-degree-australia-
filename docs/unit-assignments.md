# Unit Assignment Register

This register prevents two contributors from drafting the same unit. **Claim a
unit by adding your handle here (in a small PR) and opening a "New Unit Draft"
issue** before you start writing. See
[`docs/contributor-onboarding.md`](contributor-onboarding.md) for the full flow.

- **Unassigned** — open to claim
- A GitHub handle in *Assigned to* means that unit is taken; coordinate with that
  person before starting work on it.
- When a unit is **Published**, leave the final author(s) recorded here.

> Statuses mirror [`STATUS.md`](../STATUS.md); keep the two consistent in the same PR.

### How to claim

1. Pick a ⬜ Planned unit from [`STATUS.md`](../STATUS.md).
2. Add your handle to the relevant row here in a small PR (e.g. break a major's
   row into per-unit rows as units are claimed).
3. Open a **New Unit Draft** issue for the unit.
4. Start authoring against `templates/unit-template.md`.
